Pitch:
"Hazard Haul" is a sci-fi auto shooter game where players must survive waves of enemies across 5 different planets. Each planet presents unique challenges, and failure means retrying from the start. The game is downloadable on itch.io.

What i did:
• Used Unity Engine with multiple Synty asset packs (Synty Sci-Fi City and others)
• Created a custom Timer System hooked directly into Unity's PlayerLoop
• Implemented delegates and event-driven communication systems for Scene switching, UI and Saving
• Applied generics for flexible, reusable code architecture (Generic singleton class for example)
• Utilized Unity NavMesh for AI pathfinding and enemy navigation

What I Learned:
• Solid understanding of C# generics and delegates
• Understanding of what is PlayerLoop system and what custom game systems we could inject there
• Fundamentals of OOP architecture and why to apply SOLID principles
• Shader programming basics and visual effects creation
• Event-driven architecture patterns

Next Learning Goals:
• LINQ for more efficient data queries
• Dependency Injection patterns
• Level design principles and playtesting methodologies
• Unity Editor API to create custom tools and solutions
• More shader programming techniques

Final Reflection:
This school project was developed solo as Portfolio Project 1 for my game development studies at Metropolia UAS. It taught me the critical balance between rapid prototyping and sustainable development practices, while solidifying my understanding of Unity game development fundamentals. The experience of building a complete game loop from scratch was invaluable for my growth as a developer. It was also eye-opening project of how difficult game development as a solo is.
